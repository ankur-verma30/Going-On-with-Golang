###########################
🔷 Introduction to Golang
###########################
Go (also called Golang) is:
    ✔️ A statically typed
    ✔️ compiled
    ✔️ concurrent
    ✔️ garbage-collected
programming language.

Designed to:
    ✔️ Build fast
    ✔️ scalable
    ✔️ reliable
    ✔️ production-grade
systems with simple code


🔁 Who Created Go & Why?
👨‍💻 Creators (Google, 2007)
1️⃣ Robert Griesemer
2️⃣ Rob Pike
3️⃣ Ken Thompson (creator of Unix & C)


❓ Why Go Was Create
Google engineers face➤ d problems with:
➤ Slow compile times in C++ ❌
➤ Complex syntax and abstractions ❌
➤ Poor concurrency support ❌
➤ Hard-to-maintain large codebases ❌

👉 Goal of Go:
“A language as fast as C, as readable as Python, and as safe as Java — but simpler.”


🧠 Go Design Philosophy
Go follows strong opinions:
| Principle                    | Meaning                           |
| ---------------------------- | --------------------------------- |
| Simplicity                   | Less syntax, fewer keywords       |
| Readability                  | Code should be readable by anyone |
| Explicitness                 | No hidden magic                   |
| Composition over inheritance | Interfaces + structs              |
| Concurrency as first-class   | Built into the language           |
| Fast builds                  | Designed for large codebases      |


🏗 Go Architecture (VERY IMPORTANT)
🔁 High-Level Go Architecture
Go Source Code (.go)
        ↓
      Compiler
        ↓
     Machine Code
        ↓
   Go Runtime (GC, Scheduler)
        ↓
     Operating System


🔁 1. Go Compiler
    ✔️ Ahead-of-Time (AOT) compiled
    ✔️ No JVM / no bytecode
    ✔️ Produces single native binary

Advantages
    ✔️ Fast startup 🚀
    ✔️ No runtime dependency
    ✔️ Easy deployment

Example:
go build main.go
./main


🔁 2. Go Runtime
Unlike Java, Go runtime is small and focused.
Go Runtime Manages:
    ✔️ Goroutines scheduling
    ✔️ Garbage Collection
    ✔️ Memory allocation
    ✔️ Channel communication
    ✔️ Stack management
⚠️ You don’t manage runtime directly — Go does.


🔁 3. Go Scheduler (GMP Model)
The Famous G-M-P Model
| Component | Meaning                 |
| --------- | ----------------------- |
| G         | Goroutine               |
| M         | OS Thread               |
| P         | Processor (logical CPU) |
G (goroutine)
   ↓
P (scheduler)
   ↓
M (OS thread)


❓ Why this matters?
➤ Millions of goroutines can run on few OS threads
➤ Extremely lightweight concurrency
➤ No thread explosion


🔁 4. Memory Architecture
Stack vs Heap in Go
➤ Goroutines start with small stack (~2KB)
➤ Stack grows/shrinks dynamically
➤ Escape analysis decides:
    ✔️ Stack allocation ✅
    ✔️ Heap allocation ❌
👉 Improves performance significantly.


⭐ Features of Go Language (In-Depth)
1️⃣ Simple & Clean Syntax
No
    ✔️ Classes 
    ✔️ Inheritance 
    ✔️ Generics (added later, minimal) ❌
    ✔️ Exceptions ❌

Example:
if err != nil {
    return err
}
✔️ Explicit error handling = clarity.


2️⃣ Static Typing with Type Inference
var a int = 10
b := 20
✔️ Type safety
✔️ Compiler catches bugs early


3️⃣ Concurrency Built-In (BIGGEST FEATURE)
Goroutines
go doWork()

➤ Lightweight
➤ Managed by Go runtime
➤ Cheap compared to threads

Channels
ch := make(chan int)
ch <- 10

➤ Safe communication
➤ Avoid shared memory
➤ “Don’t communicate by sharing memory, share memory by communicating.”


4️⃣ Garbage Collection (Low-Latency)
➤ Concurrent GC
➤ Optimized for:
    ✔️ Low pause time
    ✔️ High throughput
➤ Used in real-time systems like servers.


5️⃣ Interfaces (Go’s Superpower)
Implicit Implementation
type Writer interface {
    Write([]byte)
}
If a struct has Write() → it implements Writer
No implements keyword 🔥


6️⃣ Composition over Inheritance
type Engine struct {}
type Car struct {
    Engine
}
➤ Avoids fragile inheritance
➤ More flexible design


7️⃣ Standard Library (Very Rich)
Includes:
    ✔️ HTTP server
    ✔️ JSON
    ✔️ Cryptography
    ✔️ Concurrency utilities
    ✔️ File I/O
    ✔️ Testing
Example:
http.ListenAndServe(":8080", nil)
No framework required.


8️⃣ Cross-Platform Compilation
GOOS=linux GOARCH=amd64 go build
✔️ Build once
✔️ Run anywhere


9️⃣ Built-in Tooling
| Tool    | Purpose         |
| ------- | --------------- |
| go fmt  | Code formatting |
| go test | Testing         |
| go mod  | Dependency mgmt |
| go vet  | Static analysis |
